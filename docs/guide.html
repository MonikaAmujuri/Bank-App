Guide

This documentation contains the API reference for the Bank App project.

Endpoints are organized into separate pages for Auth, Admin, Agent, Customer, and Loans.

Modules

Auth: Login and authentication APIs.

Admin: Admin dashboard, agents, users, and profile APIs.

Agent: Agent dashboard, user management, and profile APIs.

Customer: Customer profile, documents, and customer loan APIs.

Loans: Loan controller APIs for admin, agent, and customer workflows.

Base URL

http://localhost:5000

Authorization Format

Authorization: Bearer <token>

API Conventions

Protected routes require a valid JWT token in the Authorization header.

File upload APIs use formData in the request body.

Filter-based APIs use query parameters such as ?status=approved or ?type=Home%20Loan.

Dynamic route values like :id, :userId, and :loanId must be replaced with actual values.

Notes

All examples in this documentation are written using Fetch API.

Responses shown here are sample outputs and may vary depending on the database data.

Role-based access applies to Admin, Agent, and Customer endpoints.
