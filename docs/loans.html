Loan APIs

This section contains loan-related endpoints used by the admin and agent to create, update, review, and manage loans.

Start Loan

This endpoint allows the agent to start a new loan draft for a selected user.

POST /api/loans/start 
fetch(
  "http://localhost:5000/api/loans/start",
  {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${token}`,
    },
    body: JSON.stringify({ userId }),
  }
);
👇 Output 
{
  "message": "Loan draft created",
  "loan": {
    "loanId": "LOAN0008",
    "status": "draft"
  }
}

Important: This endpoint requires a valid agent JWT token. The request body must contain the selected user ID, and the backend creates a new loan in draft status.

Update Loan - Loan Details

Role: Agent

This endpoint allows the agent to update a loan section such as loan details by loan ID.

PUT /api/loans/:loanId 
fetch(`http://localhost:5000/api/loans/${loanId}`, {
  method: "PUT",
  headers: {
    "Content-Type": "application/json",
    Authorization: `Bearer ${token}`,
  },
  body: JSON.stringify({
    section,
    data: updatedData,
  }),
});
👇 Output 
{
  "message": "loanDetails updated successfully",
  "loanId": "LOAN0008"
}

Important: This endpoint requires a valid agent JWT token. Replace :loanId with the actual loan ID, and send the section name plus updated section data in the request body.

Reject Loan

Role: Agent

This endpoint allows the agent to reject a loan by loan ID and send remarks or reason in the request body.

PUT /api/loans/:loanId/reject 
fetch(`http://localhost:5000/api/loans/${loanId}/reject`, {
  method: "PUT",
  headers: {
    "Content-Type": "application/json",
    Authorization: `Bearer ${token}`,
  },
  body: JSON.stringify({ remarks: reason }),
});
👇 Output 
{
  "message": "Loan rejected successfully"
}

Important: This endpoint requires a valid agent JWT token. Replace :loanId with the actual loan ID and send the rejection reason in the request body.
